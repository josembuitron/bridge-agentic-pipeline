The Five Phases

Phase 1: Translate

BRIDGE B-R-I-D analysis. Extracts business challenges, root causes, impact metrics, and data context from raw input.

Sonnet Ojo Critico

Output: Technical Definition + BRIDGE Analysis

Phase 2: Research

Live documentation research. APIs, platforms, and technologies investigated using 6-tier doc access chain.

Sonnet Ojo Critico

Output: Research Report with verified tech stack

Phase 3: Architect

Complete solution design: architecture diagrams, cloud cost models, file manifest, specialist team, vertical slices.

Opus Ojo Critico Plan Checker

Output: Solution Proposal + Mermaid/Excalidraw diagrams

Phase 4: Build

Dynamic specialist agents execute vertical slices using TDD. Walking skeleton first, then incremental hardening. Per-slice security scans.

Sonnet Semgrep per-slice

Output: Working code + tests + build manifest

Phase 5: Validate & Deliver

Goal-backward verification, 6-pass code review, SAST security scanning, secrets detection, dependency audit, quality scoring. Up to 22 Trail of Bits security skills activated.

Opus BLOCKING security gate 6-pass PR review

Output: Validation report + sanitized client deliverables

Phase 4: Build Process Detail

Pre-Phase Skill Cache

  • superpowers:test-driven-development
  • sharp-edges (ToB)
  • property-based-testing (if critical logic)
  • testing-handbook-skills (if critical logic)
  • frontend-design (if frontend)
  • building-secure-contracts (if blockchain)

Dynamic Agent Creation

Dependency resolution before spawn:

  • CLI tools → auto-install via setup script
  • npm/pip packages → install before spawning
  • MCP servers → add to tools or degrade gracefully
  • ToB skills → invoke and embed methodology
  • Scripts → create mock servers, data generators

Quality check before writing agent:

  • Clear, specific task (not vague)
  • ALL tools assigned
  • Doc access chain included
  • Prompt < 750 words
  • Completion signal defined

Dev-QA Loop Per Slice

BUILD (TDD: red → green → refactor)

↓
TEST (vitest / pytest / framework-specific)

↓
HARDEN (error paths, boundaries, concurrency)

↓
E2E (Playwright smoke test if frontend)

↓
SECURITY SCAN (semgrep per-slice)

↓
BRIDGE_SLICE_COMPLETE

Rules: Max 3 retries per slice. Slice 1 failure = architecture wrong (escalate immediately). Each retry includes failure reason from previous attempt.

Stall Detection

  • Missing BRIDGE_SLICE_COMPLETE signal → stall
  • Error keywords ("failed", "cannot") → stall
  • No output / maxTurns hit → timeout stall
  • Same slice stalls 2x → auto-escalate

Phase 5: Validation & Security Detail

Validation Agents (4 sequential)

5.1a Validator — Goal-backward verification. Stub detection. Requirements traceability. BRIDGE alignment. Verdict: APPROVE/REJECT

5.1b Code Reviewer — Clean code, test quality, YAGNI, documentation. ESLint scan. Verdict: PASS/FAIL

5.1c Security Auditor (BLOCKING) — SAST (semgrep), secrets (gitguardian), dependency audit (npm audit/pip-audit), OWASP Top 10, insecure defaults. Verdict: SECURE/BLOCKED

5.1d PR Review (6-pass) — Code review, test analysis, silent failure hunting, type design, comment accuracy, code simplification

Quality Score

requirements_coverage × 0.35 + test_pass_rate × 0.25 + security_score × 0.20 + code_quality × 0.10 + documentation × 0.10

≥ 0.80 APPROVE | 0.60-0.79 CONDITIONAL | < 0.60 REJECT

Security Skills Activated in Phase 5

Always:

  • static-analysis (CodeQL + Semgrep + SARIF)
  • supply-chain-risk-auditor
  • differential-review (code vs architecture drift)
  • spec-to-code-compliance
  • audit-context-building
  • fp-check (false positive verification)

Conditional:

  • variant-analysis (if vulnerability found)
  • semgrep-rule-creator (if vulnerability found)
  • agentic-actions-auditor (if GitHub Actions)
  • zeroize-audit (if crypto/secrets)
  • constant-time-analysis (if timing-sensitive)
  • building-secure-contracts (if blockchain)
  • firebase-apk-scanner (if Android+Firebase)
  • second-opinion (if external LLM CLI)

Security Gate

BLOCKING (default): ANY critical finding = BLOCKED. User must accept risk per finding explicitly.

Advisory: Findings logged, do not block.

Resilience & Context Management

Context Budget Protocol (9 Rules)

  • R1: Never accumulate — write to disk, keep 3-5 line summary
  • R2: Phase handoff via disk, not context
  • R3: Specialist prompts < 2,000 tokens
  • R4: Orchestrator keeps only checkpoint summaries
  • R5: Large project detection (>5 specialists)
  • R6: Context-by-reference pattern for all agents
  • R7: Phase-based context refresh (before Phase 4, every 2 specialists, before Phase 5)
  • R8: Prompt size guard (<750 words, extract to temp file if larger)
  • R9: Emergency recovery (detect degradation, re-read core.md + state.json)

Cost Tracking

  • Per-agent estimation after every spawn (chars ÷ 4)
  • Optional budget cap with 80% warning, 100% pause
  • Model pricing: Opus $15/$75, Sonnet $3/$15, Haiku $0.80/$4 per 1M tokens

Pipeline State & Resumability

  • state.json tracks: current phase, completed phases, specialist status, last checkpoint
  • Resume reads state.json first, falls back to file scanning
  • Git tags after each phase for rollback

Behavioral Guardrails

  • Tool denial: Try fallback chain, never stop
  • Analysis paralysis: 5+ reads without writing = must explain or report BLOCKED
  • Deviation rules: Auto-fix bugs; escalate architecture; skip scope creep
  • Stall detection: Missing signals, error keywords, timeouts
  • Rejection loops: Max 3 retries per slice, max 2 validation cycles

Cross-Run Learning

  • Failures requiring 2+ attempts generate lessons
  • Client knowledge graph persists across projects (strict isolation)
  • Successful specialists archived for future reuse

Get Started

# Clone
git clone https://github.com/josembuitron/bridge-agentic-pipeline.git
cd bridge-agentic-pipeline

# Setup (see SETUP.md)
pip install -U crawl4ai && crawl4ai-setup
pip install semgrep

# Run
claude
/bridge

Human approval gates at every phase. Stop at any point and get deliverables. Most common exit: Phase 3 (client proposals without writing code).
