# Solution Architecture Skill

When designing solutions:

1. Start from the Technical Definition requirements and Research Report findings
2. Design components with clear separation of concerns
3. Use Mermaid diagrams for all visual representations
4. Create a complete file manifest with every file needed
5. Specify the exact specialist agents needed with their roles, tasks, and tools
6. Define execution groups with dependency ordering
7. Include deployment and testing strategies
