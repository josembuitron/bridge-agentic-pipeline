# Requirements Translation Skill

When translating business requirements, follow these steps:

1. Read the input thoroughly - identify all explicit and implicit requirements
2. Categorize: functional, non-functional, integration, data, security, performance
3. Prioritize: HIGH (must have), MEDIUM (should have), LOW (nice to have)
4. Identify all systems, platforms, APIs, and databases mentioned
5. Extract constraints: timeline, budget, technology restrictions, compliance
6. Flag ambiguities that need human clarification
7. Output using the Technical Definition template format

Always number requirements as REQ-001, REQ-002, etc.
Use MUST/SHOULD/MAY language for priority indication.
